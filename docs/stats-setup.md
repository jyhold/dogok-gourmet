# 📊 관리자 통계 대시보드 설정 (기획서 §11)

`/stats` 에서 방문자·룰렛·좋아요 통계를 봅니다. **순서대로** 하세요 — 1번을 건너뛰면 관리자DB가 오염됩니다.

---

## ⚠️ 먼저 읽기

기존 Apps Script는 **허용 목록에 없는 탭이 오면 에러 대신 `restaurants`에 그냥 씁니다.**
이 상태로 통계를 켜면 **누가 앱에 들어올 때마다 restaurants 탭에 이벤트 행이 쌓입니다.**
그래서 아래 1~3을 마치고 **4번 셀프테스트를 통과한 뒤에야** 수집을 켭니다.

코드에도 안전장치가 있습니다: 웹훅이 `stats`가 아닌 곳에 썼다고 응답하면 **즉시 기록을 중단**하고 대시보드에 경고를 띄웁니다.

---

## 1) Apps Script 교체 (필수)

구글 시트 → **확장 프로그램 → Apps Script** → `doPost`를 [sheet-sync-setup.md](sheet-sync-setup.md)의
최신 버전으로 교체합니다. 핵심 변경 두 가지:

```javascript
const ALLOWED = ['restaurants', 'coffee', 'stats'];   // ← stats 추가
...
const name = body.sheet || 'restaurants';
if (ALLOWED.indexOf(name) < 0) return json({ error: '허용되지 않은 탭: ' + name });  // ← 조용한 폴백 제거
```

그리고 반드시 **배포 → 새 배포**(기존 배포 수정이 아니라 새 버전). 이걸 안 하면 코드를 고쳐도 옛 버전이 돕니다.

## 2) `stats` 탭 만들기

시트 하단 **+** → 탭 이름을 정확히 **`stats`** 로. 1행에 헤더 7개를 A1부터 붙여넣기:

```
ts	event	visitor	mode	place	category_sub	detail
```

> 열 서식은 그대로 둬도 됩니다. `ts`는 `20260715-102345` 형태라 시트가 날짜로 바꾸지 않습니다.
> (`2026-07-15T10:23Z` 같은 형식을 쓰면 시트가 날짜 셀로 바꿔버려 되읽을 때 깨집니다.)

## 3) 환경변수 2개 추가

Vercel → 프로젝트 → **Settings → Environment Variables**:

| 이름 | 값 | 설명 |
|---|---|---|
| `STATS_KEY` | 임의의 긴 문자열 | `/stats` 페이지 비밀번호. 예: `dogok-stats-7f3k…` |
| `STATS_ENABLED` | `FALSE` | **일단 FALSE로.** 4번 통과 후 TRUE로 바꿉니다 |

Production/Preview/Development 모두 체크 → **Save → 재배포**.

## 4) 셀프테스트 (켜기 전 확인)

1. `https://dogok-gourmet.vercel.app/stats` 접속 → `STATS_KEY` 입력
2. **🔧 수집 셀프테스트** 클릭

- ✅ **`stats 탭에 정상 기록됨`** → 5번으로. (시트 stats 탭에 `(셀프테스트)` 행이 생깁니다. 지워도 됩니다)
- ✖ **`'restaurants' 탭에 쓰였습니다`** → 1번이 안 끝난 것입니다.
  **restaurants 탭 마지막 행을 삭제**하고, Apps Script를 새 배포한 뒤 다시 시도하세요.

## 5) 수집 켜기

`STATS_ENABLED` 를 **`TRUE`** 로 바꾸고 재배포. 이때부터 기록이 쌓입니다.

> 대시보드 상단에 `⚠️ 수집이 꺼져 있습니다` 배너가 사라지면 정상입니다.

---

## 무엇이 기록되나

| 이벤트 | 언제 | 담기는 것 |
|---|---|---|
| `visit` | 첫 진입 (세션 1회) | 익명 ID |
| `spin` | 룰렛 당첨 | 모드·가게·카테고리·필터·재추첨 여부 |
| `like` | 결과 카드 **👍 좋아요** | 모드·가게 |
| `map` | **네이버지도에서 보기** 클릭 | 모드·가게 |

- **개인정보는 수집하지 않습니다.** 방문자 구분은 브라우저 localStorage에 저장한 난수(`v-x8k2…`)뿐이고,
  IP·UA·위치는 저장하지 않습니다. 사용자가 지우면 새 방문자로 잡히는 느슨한 식별자입니다.
- **로컬 개발(`npm run dev`)에서는 기록되지 않습니다** — 개발 트래픽이 통계를 오염시키지 않도록 막아뒀습니다.
- `USE_MOCK=TRUE` 면 대시보드가 가짜 데이터를 보여줍니다(키 없이 화면 개발용). 상단에 🧪 배너가 뜹니다.

## 알아둘 한계

- **시트는 이벤트 로그 전용 저장소가 아닙니다.** 하루 수백 건까지가 안전선입니다.
  더 커지면 Vercel KV(Redis) 같은 곳으로 옮겨야 합니다.
- **과거 데이터는 없습니다.** 켠 시점부터 쌓입니다.
- JS를 실행하는 브라우저만 기록되므로 크롤러는 자연히 빠지지만, 정확한 UV는 아닙니다.
- 대시보드는 1분 캐시입니다. 방금 남긴 기록이 안 보이면 **↻ 새로고침**을 누르세요.
